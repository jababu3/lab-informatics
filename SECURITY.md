# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.x     | Yes       |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please **do not open a public GitHub issue**.

Instead, report it privately via one of the following:

- **GitHub Security Advisories:** Use the [Report a vulnerability](https://github.com/jababu3/lab-informatics/security/advisories/new) button on the Security tab of this repository.
- **Email:** Contact the maintainer directly through GitHub ([@jababu3](https://github.com/jababu3)).

### What to include

- A description of the vulnerability and its potential impact
- Steps to reproduce or a proof-of-concept (if available)
- The version(s) affected

### What to expect

- Acknowledgement within **3 days**
- A status update within **2 weeks**
- Credit in the release notes if you wish

## Scope

This project is a **learning and demonstration LIMS application**. It is not intended for production use without additional hardening (see `README.md`). Security reports related to the demo environment defaults (e.g., default credentials in `.env.example`) are appreciated but noted as by-design for local development.
